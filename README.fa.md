# اسکنر DNSTT

**[English / انگلیسی](README.md)**

یک اسکنر موازی DNS که سرورهای DNS را از نظر اتصال از طریق تونل‌زنی [dnstt](https://www.bamsoftware.com/software/dnstt/) تست می‌کند. این ابزار بررسی می‌کند که آیا هر سرور DNS می‌تواند یک تونل dnstt برقرار کند و آن‌ها را بر اساس وضعیت دسته‌بندی می‌کند.

## نحوه عملکرد

1. لیست آی‌پی سرورهای DNS را از یک فایل (فرمت CSV یا TXT) می‌خواند
2. برای هر سرور DNS، یک ورکر موازی اجرا می‌کند که:
   - یک نمونه `dnstt-client` را به سمت سرور DNS راه‌اندازی می‌کند
   - یک اتصال SSH از طریق تونل تلاش می‌کند
   - هر دو فرآیند را برای نشانه‌های موفقیت/شکست نظارت می‌کند
3. هر سرور DNS را در یکی از چهار وضعیت دسته‌بندی می‌کند:
   - **CLEAN (تمیز)** - تونل کار می‌کند، دست‌دهی SSH انجام شد (DNS قابل استفاده است)
   - **BLOCKED_BY_DPI (مسدود توسط DPI)** - استریم تونل ایجاد شد اما SSH شکست خورد (احتمالاً بازرسی عمیق بسته)
   - **DEAD (مرده)** - هیچ تونل یا اتصالی برقرار نشد
   - **CRASH (خراب)** - فرآیند dnstt-client بلافاصله از کار افتاد
4. تمام سرورهای DNS سالم/کارآمد را در یک فایل خروجی با برچسب زمانی ذخیره می‌کند

## پیش‌نیازها

- سیستم‌عامل macOS (تست شده روی ARM64) یا Linux
- کلاینت SSH (به صورت پیش‌فرض روی macOS و اکثر توزیع‌های لینوکس نصب است)
- یک سرور dnstt در حال اجرا به همراه کلید عمومی آن

git clone <repository-url>

### قدم ۲: دانلود باینری dnstt-client

باینری `dnstt-client` برای تست تونل‌زنی DNS مورد نیاز است. باید نسخه مناسب سیستم‌عامل و معماری خود را دانلود کنید.

**روش الف: دانلود باینری از پیش ساخته شده**

به صفحه رسمی dnstt بروید و باینری مناسب را دانلود کنید:

https://www.bamsoftware.com/software/dnstt/

**روش ب: ساخت از سورس**

اگر باینری از پیش ساخته شده‌ای برای پلتفرم شما موجود نیست، می‌توانید خودتان بسازید. نیاز به نصب [Go](https://go.dev/dl/) دارید:

</div>

```bash
git clone https://www.bamsoftware.com/git/dnstt.git
cd dnstt/dnstt-client
go build
```

<div dir="rtl">

این دستور یک باینری `dnstt-client` در پوشه فعلی ایجاد می‌کند.

**باینری را کجا قرار دهید:**

باینری را به پوشه اصلی پروژه (همان پوشه‌ای که `dns-scanner.sh` در آن قرار دارد) منتقل یا کپی کنید:

</div>

```bash
cp dnstt-client /path/to/dnsscanner/
```

<div dir="rtl">

پوشه پروژه شما باید به این شکل باشد:

</div>

```
dnsscanner/
├── dns-scanner.sh
├── dnstt-client          <-- باینری را اینجا قرار دهید
├── .env
├── .env.example
├── dns-list.txt
└── README.md
```

<div dir="rtl">

مطمئن شوید که باینری قابل اجرا است:

</div>

```bash
chmod +x dnstt-client
```

<div dir="rtl">

### قدم ۳: پیکربندی محیط

۱. فایل نمونه محیط را کپی کنید:

</div>

```bash
cp .env.example .env
```

<div dir="rtl">

۲. فایل `.env` را در یک ویرایشگر متن باز کنید و مقادیر خود را وارد کنید:

</div>

```bash
nano .env
```

<div dir="rtl">

توضیح هر متغیر:

| متغیر | الزامی | توضیحات |
|---|---|---|
| `PUB_KEY` | بله | کلید عمومی سرور dnstt شما. این را از کسی که سرور را راه‌اندازی کرده دریافت کنید. |
| `DNSTT_DOMAIN` | بله | نام دامنه پیکربندی شده برای تونل‌زنی dnstt (مثلاً `t.example.com`). |
| `SSH_USER` | بله | نام کاربری SSH روی سرور تونل (معمولاً `root`). |
| `DNSTT_BIN` | بله | مسیر باینری dnstt-client. اگر در پوشه پروژه قرار دادید، از `./dnstt-client` استفاده کنید (یا `./dnstt-client-darwin-arm64` اگر نام فایل این است). |
| `DNS_LIST_FILE` | بله | مسیر فایل لیست سرورهای DNS شما (`.csv` یا `.txt`). |
| `OUTPUT_FILE` | خیر | نام فایل خروجی سفارشی. اگر تنظیم نشود، یک فایل با برچسب زمانی مثل `working-dns_2026-03-04_16-30-00.txt` به صورت خودکار ایجاد می‌شود. |
| `MAX_WAIT_TIME` | خیر | چند ثانیه برای هر بررسی DNS صبر کند قبل از انصراف (پیش‌فرض: `45`). |
| `BASE_PORT` | خیر | شماره پورت محلی شروع برای فرآیندهای ورکر (پیش‌فرض: `10000`). |
| `MAX_PARALLEL` | خیر | چند سرور DNS به صورت همزمان تست شوند (پیش‌فرض: `5`). برای اسکن سریع‌تر افزایش دهید، برای منابع محدود کاهش دهید. |

### قدم ۴: آماده‌سازی لیست DNS

یک فایل با آی‌پی سرورهای DNS که می‌خواهید تست کنید بسازید. دو فرمت پشتیبانی می‌شود:

**فرمت TXT** - هر آی‌پی در یک خط، کامنت با `#` پشتیبانی می‌شود:

</div>

```
8.8.8.8
1.1.1.1
# This is a comment
9.9.9.9
```

<div dir="rtl">

**فرمت CSV** - آی‌پی در ستون اول، ردیف اول به عنوان هدر در نظر گرفته شده و نادیده گرفته می‌شود:

</div>

```csv
ip,provider,country
8.8.8.8,Google,US
1.1.1.1,Cloudflare,US
```

<div dir="rtl">

این فایل را ذخیره کنید و مطمئن شوید که `DNS_LIST_FILE` در فایل `.env` شما به آن اشاره می‌کند.

### قدم ۵: اجرای اسکنر

</div>

```bash
chmod +x dns-scanner.sh
./dns-scanner.sh
```

<div dir="rtl">

اسکنر هر سرور DNS را تست می‌کند و نتایج را در ترمینال با وضعیت‌های رنگی نشان می‌دهد. هر اسکن یک فایل خروجی جدید با برچسب زمانی ایجاد می‌کند (مثلاً `working-dns_2026-03-04_16-30-00.txt`) تا نتایج قبلی شما همیشه حفظ شوند.

## نمونه خروجی

</div>

```
==================================================
  DNSTT Scanner (Parallel | Max 5 workers)
==================================================
  DNS list : dns-list.txt (3 servers)
  Output   : working-dns_2026-03-04_16-30-00.txt
==================================================

 [ CLEAN ] 8.8.8.8
 [ BLOCKED BY DPI ] 1.1.1.1
 [ DEAD ] 9.9.9.9

==================================================
  Done! 1/3 servers are clean.
  Working DNS saved to: working-dns_2026-03-04_16-30-00.txt
==================================================
```

## عیب‌یابی

| مشکل | راه‌حل |
|---|---|
| `Error: .env file not found` | دستور `cp .env.example .env` را اجرا کنید و مقادیر خود را وارد کنید. |
| `Error: PUB_KEY is not set` | فایل `.env` را باز کنید و مطمئن شوید تمام متغیرهای الزامی پر شده‌اند. |
| `Permission denied` هنگام اجرای اسکریپت | دستور `chmod +x dns-scanner.sh` را اجرا کنید. |
| `dnstt-client: command not found` یا خرابی | مطمئن شوید باینری در مسیری که در `DNSTT_BIN` مشخص کردید وجود دارد و قابل اجرا است (`chmod +x`). |
| همه سرورها DEAD نشان داده می‌شوند | اتصال اینترنت خود را بررسی کنید و صحت `DNSTT_DOMAIN` و `PUB_KEY` را تأیید کنید. |
| اسکن‌ها خیلی کند هستند | `MAX_PARALLEL` را در `.env` افزایش دهید (مثلاً `10`)، یا `MAX_WAIT_TIME` را کاهش دهید. |
