# اسکنر DNSTT

**[English](README.md)**

این ابزار یک اسکنر موازی است که وضعیت اتصال سرورهای DNS را برای استفاده در
تونل‌های [dnstt](https://www.bamsoftware.com/software/dnstt/) بررسی می‌کند.
اسکنر بررسی می‌کند که آیا هر سرور DNS می‌تواند با موفقیت یک تونل ارتباطی بسازد
یا خیر، و سپس آن‌ها را بر اساس وضعیت دسته‌بندی می‌کند.

## نحوه کارکرد

۱. یک لیست از آی‌پی‌های سرور DNS را از یک فایل (با فرمت CSV یا TXT) می‌خواند. ۲.
برای هر سرور DNS، یک پردازش (Worker) موازی اجرا می‌کند که کارهای زیر را انجام
می‌دهد:

- یک نمونه از کلاینت `dnstt-client` را متصل به آن سرور DNS اجرا می‌کند.
- تلاش می‌کند از طریق تونل ایجاد شده، یک ارتباط SSH برقرار کند.
- هر دو پردازش را مانیتور می‌کند تا از موفقیت یا شکست ارتباط مطلع شود. ۳. هر
  سرور DNS را در یکی از چهار وضعیت زیر دسته‌بندی می‌کند:
- **CLEAN (سالم)** - تونل به درستی کار می‌کند و ارتباط SSH برقرار شده است (این
  DNS قابل استفاده است).
- **BLOCKED_BY_DPI (مسدود شده)** - تونل اولیه ایجاد شده اما ترافیک SSH عبور
  نکرده است (احتمالاً به دلیل سیستم فیلترینگ و DPI).
- **DEAD (مُرده)** - هیچ تونل یا ارتباطی با سرور برقرار نشد.
- **CRASH (کرش)** - کلاینت dnstt در همان لحظه اجرا دچار خطا شده و بسته شده است.
  ۴. در نهایت، تمام DNSهای سالم و قابل استفاده را در یک فایل خروجی جدید (همراه
  با تاریخ و ساعت) ذخیره می‌کند.

## پیش‌نیازها

- سیستم‌عامل مک (تست شده روی پردازنده‌های ARM64) یا لینوکس.
- کلاینت SSH (به صورت پیش‌فرض روی مک و اکثر توزیع‌های لینوکس نصب است).
- یک سرور dnstt فعال به همراه کلید عمومی (Public Key) آن.

## راه‌اندازی گام‌به‌گام

### گام اول: دریافت کدهای پروژه (Clone)

```bash
git clone <repository-url>
cd dnsscanner

```

### گام دوم: دانلود فایل اجرایی dnstt-client

برای تست تونل DNS، به فایل اجرایی (Binary) برنامه کلاینت نیاز دارید. باید نسخه
از پیش کامپایل شده و مناسبِ سیستم‌عامل خود را از سایت زیر دانلود کنید:

**لینک دانلود:** https://dnstt.network/

_نکته مهم: اسم فایلی که دانلود می‌کنید بر اساس سیستم‌عامل و پردازنده شما متفاوت
است (مثلاً برای لپ‌تاپ‌های مک `dnstt-client-darwin-arm64` و برای سرورهای لینوکس
`dnstt-client-linux-amd64` است)._

**فایل اجرایی را کجا قرار دهیم؟**

فایل دانلود شده را کپی کرده و دقیقاً در مسیر اصلی پروژه (کنار فایل
`dns-scanner.sh`) قرار دهید:

```bash
# مثال برای سیستم‌عامل مک
cp dnstt-client-darwin-arm64 /path/to/dnsscanner/

```

ساختار پوشه پروژه شما باید شبیه به این باشد:

```
dnsscanner/
├── dns-scanner.sh
├── dnstt-client-darwin-arm64    <-- فایل اجرایی را اینجا قرار دهید
├── .env
├── .env.example
├── dns-list.txt
└── README.md

```

فراموش نکنید که دسترسی اجرا (Executable) را به فایل بدهید:

```bash
# به جای نام زیر، اسم دقیق فایل دانلودی خودتان را بگذارید
chmod +x dnstt-client-darwin-arm64

```

### گام سوم: پیکربندی متغیرهای محیطی

۱. از روی فایل نمونه، فایل کانفیگ اصلی را بسازید:

```bash
cp .env.example .env

```

۲. فایل `.env` را در یک ویرایشگر متنی باز کنید و اطلاعات خود را وارد کنید:

```bash
nano .env

```

توضیحات مربوط به هر متغیر:

| متغیر           | اجباری | توضیحات                                                                                                                                                    |
| --------------- | ------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `PUB_KEY`       | بله    | کلید عمومی (Public Key) سرور dnstt شما.                                                                                                                    |
| `DNSTT_DOMAIN`  | بله    | دامنه‌ای که برای تونل dnstt تنظیم کرده‌اید (مثلاً `t.example.com`).                                                                                        |
| `SSH_USER`      | بله    | نام کاربری SSH در سرور مقصد (معمولاً `root` است).                                                                                                          |
| `DNSTT_BIN`     | بله    | مسیر فایل اجرایی dnstt-client. **مطمئن شوید این مقدار دقیقاً برابر با نام فایلی است که در گام دوم دانلود کرده‌اید** (مثلاً `./dnstt-client-darwin-arm64`). |
| `DNS_LIST_FILE` | بله    | مسیر فایل متنی حاوی لیست سرورهای DNS (فرمت `.csv` یا `.txt`).                                                                                              |
| `OUTPUT_FILE`   | خیر    | نام فایل خروجی. اگر خالی بماند، اسکریپت خودکار یک فایل با تاریخ و ساعت دقیق می‌سازد.                                                                       |
| `MAX_WAIT_TIME` | خیر    | حداکثر زمان انتظار (به ثانیه) برای تست هر DNS قبل از قطع کردن ارتباط (پیش‌فرض: `45`).                                                                      |
| `BASE_PORT`     | خیر    | پورت پایه برای اجرای پردازش‌های موازی روی سیستم لوکال (پیش‌فرض: `10000`).                                                                                  |
| `MAX_PARALLEL`  | خیر    | تعداد تست‌های همزمان (پیش‌فرض: `5`). برای سرعت بیشتر آن را افزایش دهید.                                                                                    |

### گام چهارم: آماده‌سازی لیست DNS

یک فایل شامل آی‌پی‌های سرور DNS که قصد تست کردنشان را دارید بسازید. اسکریپت از
دو فرمت پشتیبانی می‌کند:

**💡 نکته حرفه‌ای: پیدا کردن سرورهای DNS** اگر هنوز لیستی از سرورهای DNS ندارید،
می‌توانید ابتدا از ابزارهایی مثل
[PYDNS-Scanner](https://github.com/xullexer/PYDNS-Scanner) برای جستجو و پیدا
کردن آن‌ها استفاده کنید. خروجی ابزار PYDNS-Scanner یک فایل با فرمت `.csv` است.
شما می‌توانید **دقیقاً همان فایل را بدون هیچ تغییری** در مسیر اصلی این پروژه
قرار دهید و متغیر `DNS_LIST_FILE` را روی آن تنظیم کنید تا اسکریپت از آن استفاده
کند!

**فرمت TXT** - در هر خط یک آی‌پی (پشتیبانی از کامنت با کاراکتر `#`):

```
8.8.8.8
1.1.1.1
# این یک کامنت است
9.9.9.9

```

**فرمت CSV** - آی‌پی باید در ستون اول باشد (خط اول به عنوان هِدِر در نظر گرفته
شده و نادیده گرفته می‌شود):

```csv
ip,provider,country
8.8.8.8,Google,US
1.1.1.1,Cloudflare,US

```

فایل را ذخیره کنید و مطمئن شوید مسیر آن در `.env` به درستی وارد شده است.

### گام پنجم: اجرای اسکنر

```bash
chmod +x dns-scanner.sh
./dns-scanner.sh

```

اسکنر شروع به کار کرده و وضعیت اتصال هر سرور DNS را با رنگ‌بندی مشخص در ترمینال
نمایش می‌دهد. با هر بار اجرای اسکنر، یک فایل خروجی جدید (مثلاً
`working-dns_2026-03-04_16-30-00.txt`) ساخته می‌شود تا نتایج قبلی شما از بین
نروند.

## نمونه خروجی ترمینال

```
==================================================
  DNSTT Scanner (Parallel | Max 5 workers)
==================================================
  DNS list : dns-list.txt (3 servers)
  Output   : working-dns_2026-03-04_16-30-00.txt
==================================================

 [ CLEAN ] 8.8.8.8
 [ BLOCKED BY DPI ] 1.1.1.1
 [ DEAD ] 9.9.9.9

==================================================
  Done! 1/3 servers are clean.
  Working DNS saved to: working-dns_2026-03-04_16-30-00.txt
==================================================

```

## عیب‌یابی (Troubleshooting)

| مشکل                                                      | راه‌حل                                                                                                                                    |
| --------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| خطای `Error: .env file not found`                         | دستور `cp .env.example .env` را اجرا کرده و مقادیر خود را در آن وارد کنید.                                                                |
| خطای `Error: PUB_KEY is not set`                          | فایل `.env` را باز کنید و مطمئن شوید متغیرهای اجباری را پر کرده‌اید.                                                                      |
| خطای `Permission denied` هنگام اجرای اسکریپت              | دستور `chmod +x dns-scanner.sh` را اجرا کنید.                                                                                             |
| خطای `dnstt-client: command not found` یا کرش کردن کلاینت | مطمئن شوید فایل اجرایی دقیقاً در مسیر ثبت شده در `DNSTT_BIN` قرار دارد و دسترسی اجرا دارد (`chmod +x`). **اسم فایل را حتماً بررسی کنید!** |
| تمام سرورها وضعیت DEAD می‌گیرند                           | اتصال اینترنت خود را چک کنید و از صحت مقادیر `DNSTT_DOMAIN` و `PUB_KEY` مطمئن شوید.                                                       |
| سرعت اسکن خیلی پایین است                                  | مقدار `MAX_PARALLEL` را در فایل کانفیگ افزایش دهید (مثلاً `10`)، یا مقدار `MAX_WAIT_TIME` را کمی کمتر کنید.                               |
